Welcome to the dbdeploy project

The intention of dbdeploy is to simplify iterative development of databases.

Benefits

  • Consistent implementation across build tools (ant, nant, MSBuild) [CURRENTLY ONLY ANT IS SUPPORTED]
  • Consistent implementation across database management systems
  • It's easy - it just concatinates SQL scripts
  • Guaranteed read-only - dbdeploy will only perform read operations on your database
  • Sysadmin / DBA-friendly - the output is simply a SQL script
  • Generates an audit of modifications applied
  • Continuous validation of your next production upgrade - when used with a CI tool such as CruiseControl
  • Generates a corresponding undo script to roll back your changes

Overview

The recurring problem with database development is that at some point you'll need to upgrade an existing database and preserve its content. In development environments it's often possible (even desirable) to blow away the database and rebuild from scratch as often as the code is rebuilt but this approach cannot be taken forward into more controlled environments such as QA, UAT and Production.

For this reason it's necessary to maintain database modifications as a sequence of deltas.

Maintaining and making use of these deltas can quickly become a significant overhead. Issues can include:

  • Ensuring deltas get run in the correct order
  • Which database has had which delta applied?
  • How to match a tagged version of code with a specific database version?
  • How to verify that the delta scripts actually work?
  • How to verify that the undo scripts actually work?

Using dbdeploy in conjunction with SCM and CI it's possible to manage these issues painlessly.

dbdeploy plugs into your build tool (ant is currently supported, nant and MSBuild versions are coming soon) and compares a given database against a given set of delta scripts. It figures out which of the delta scripts have not yet been applied to the database and outputs a single script to migrate the database to the desired version. The script can then be run by the build tool itself, archived for later use or both.
